🐛 Bug Report

Progress Display Discrepancy Between Profile Overview & Learning Progress

Severity: Medium Platform: Web Date: April 25, 2026

Bug Summary

Inconsistent Course Completion Status Display

The Profile Overview section displays incorrect progress bars for completed courses, showing them as partially completed even when the Learning Progress page clearly indicates they are 100% completed with certificates issued. This creates confusion for users about their actual learning progress and achievement status.

Severity: Medium Impact: User Experience

Steps to Reproduce

  1. Navigate to Profile Page: Visit your HackQuest profile page (e.g., hackquest.io/user/...)
  2. View Overview Section: Click on the "Overview" tab and scroll to the "Learning Progress" section
  3. Observe Progress Bars: Note the progress bars displayed for completed courses (Ethereum, Arbitrum, Monad...)
  4. Navigate to Learning Progress Page: Click on "Learning Progress" in the left sidebar or navigate to hackquest.io/learning-progress
  5. Compare Status: Compare the completion status shown here with the Overview section
  6. Identify Discrepancy: Notice that courses marked as "Completed ✓" in Learning Progress show incomplete progress bars in the Overview section

Expected vs Actual Behavior

✅ Expected Behavior

Progress bars in Overview should match the completion status shown in Learning Progress. Completed courses should show 100% progress or display a "Completed" badge.

❌ Actual Behavior

Progress bars show partial completion (e.g., 60-80%) even when courses are fully completed with certificates issued and "Completed" status in Learning Progress.

Potential Root Causes

Technical Investigation Points:

  • Data Synchronization Issue: The Overview component may be fetching cached or stale data that hasn't been updated after course completion
  • API Endpoint Discrepancy: Overview and Learning Progress pages might be using different API endpoints that return inconsistent data
  • State Management: Progress state might not be properly propagated from the Learning Progress module to the Profile Overview component
  • Calculation Logic: The progress percentage calculation in Overview might be using a different algorithm (e.g., counting only certain types of activities vs. all completed units)
  • Database Query Issue: The query fetching progress data for Overview might have incorrect JOIN conditions or WHERE clauses
  • Client-Side Caching: Browser or app-level caching might be serving outdated progress information
  • Real-time Updates: Lack of WebSocket or polling mechanism to update progress in real-time across different sections

Recommended Solutions

  1. Unify Data Source: Ensure both Overview and Learning Progress use the same API endpoint and data calculation logic
  2. Implement Cache Invalidation: Clear or update cached progress data immediately upon course completion
  3. Add Real-time Sync: Use WebSocket or periodic polling to keep progress data synchronized across all views
  4. Progress Calculation Audit: Review and standardize the progress percentage calculation algorithm across the platform
  5. Add Visual Indicator: For completed courses, show a checkmark or "Completed" badge instead of just a progress bar
  6. Database Optimization: Ensure proper indexing and query optimization for progress-related database queries
