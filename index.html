Bug Report – GitHub Connection Failure During Profile Completion

Reported On: 13/12/2025

Open Profile Completion Page

Bug Summary

When accessing the Complete Profile section inside HackQuest Profile, the GitHub connection step behaves incorrectly.

  • The system first shows “Connected to GitHub”.
  • Immediately after, it shows an error stating GitHub is not connected.
  • However, the Profile Integrity Score section marks GitHub as successfully connected.

Due to this inconsistency, the user cannot proceed to the next profile completion step, which blocks profile onboarding.

Criticality & Impact Details

Criticality:

Medium Severity

Area of Impact:

UI / Frontend Validation Authentication Logic / OAuth Callback

How This Bug Affects User Experience

  • Prevents users from completing their onboarding/profile setup.
  • Creates confusion due to contradictory UI states (connected + error).
  • May reduce trust in profile accuracy and integration stability.
  • Breaks profile completeness score progression.

Steps to Reproduce (Validate the Bug)

  1. Go to your HackQuest Profile.
  2. Click the Complete Profile button.
  3. Navigate to the Connect GitHub section.
  4. Click Connect with GitHub and authorize the request.
  5. Observe:
    • The system briefly shows Connected.
    • Immediately it switches to an error state, claiming GitHub is not connected.
  6. Now check Profile Integrity Score.
  7. You will see GitHub listed as successfully connected.
  8. The profile completion process remains blocked.
