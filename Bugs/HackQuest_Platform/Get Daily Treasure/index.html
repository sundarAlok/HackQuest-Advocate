🐛 Bug Report

"Get Daily Treasure" Button Triggers Client-Side Error After Completing All Requirements

🔴 Severity: High 🌐 Platform: Web 📅 Date: May 2026 🎯 Feature: Daily Rewards

Bug Summary

"Get Daily Treasure" Reward Button Fails with Generic Client Error

On the HackQuest home page, after completing all required daily tasks, clicking the reward button triggers a generic client-side error without any useful debugging information.

Steps to Reproduce

  1. Open HackQuest and login
  2. Claim daily streak
  3. Like 4 projects
  4. Finish 1 unit
  5. Click "Get Daily Treasure"
  6. Error appears

Expected vs Actual

✅ Expected

Reward should be successfully granted, reflected in the user account, and visually confirmed without errors.

❌ Actual

Clicking the button results in a generic client-side error with no meaningful feedback or reward granted.

Video Demonstration

Root Causes

  • Missing Error Handling: Allows failures to surface as generic client-side crashes without useful feedback.
  • Backend API Failure: May be returning invalid or malformed responses that the frontend cannot properly process.
  • Authentication Issue: Expired or missing tokens can block the request before it completes successfully.
  • State Mismatch: Client and server conditions may be out of sync, leading to invalid reward requests.
  • Lack of Logging: Prevents capturing meaningful debugging information during failure scenarios.

Solutions

  1. Structured Error Handling: Capture failures properly and display meaningful messages instead of generic errors.
  2. API Response Validation: Ensure backend endpoints return consistent and properly formatted responses.
  3. Token Management: Validate and refresh authentication tokens before sending reward requests.
  4. Client-Side Validation: Re-check user eligibility conditions before triggering the reward claim.
  5. Error Logging Integration: Record errors with context to enable faster debugging and monitoring.

Environment

URL: https://www.hackquest.io

Browser: Chrome / Edge

OS: Windows / macOS

Notes

Daily reward failures negatively impact user trust and engagement, especially when users complete all requirements but cannot claim rewards.
