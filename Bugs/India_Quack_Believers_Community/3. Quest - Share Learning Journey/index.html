🐛 Quest Reset Freeze Bug

Task remains locked and does not reset after completion cycle

Bug Summary

After completing the quest "Share Your Web3 Learning Journey on Twitter", the task remains in a frozen state and does NOT reset even after more than one month, despite being designed as a repeatable/resetting quest.

Steps to Reproduce

  1. Complete the quest and submit required proof
  2. Wait for approval (quest gets approved ✅)
  3. Check quest status after days/weeks
  4. Even after ~1 month → task still not reset

Expected vs Actual

✅ Expected:
Quest should reset automatically after its cooldown period (monthly behavior), allowing user to complete it again.

❌ Actual:
Quest remains permanently locked/frozen after approval.
No reset, no retry option, no state change.

📸 Proof (Screenshots)

🎥 Video Proof

Possible Cause

  • Quest reset scheduler not triggering
  • Status stuck in "Approved" state without lifecycle transition
  • Missing backend cron job or reset condition
  • Frontend not refreshing quest availability state

Impact

Users cannot reattempt recurring quests → loss of engagement and rewards.
Breaks expected gamification loop.

Environment

  • Platform: HackQuest Web
  • Browser: Chrome / Edge
  • OS: Windows
